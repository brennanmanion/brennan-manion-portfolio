Analytics, Reporting & Monitoring

I was contracted to develop automated systems to build metric reports, collect data for analytics, and monitor for cataclysmic events. Each project gave me exposure to different operating systems, different Python libraries necessary to complete the task, and experience interpreting documentation for many different application programming interfaces.

My most fulfilling automation project was to automate an agency’s metric reporting. The agency would send out a PDF of their client’s most important metrics for the platforms they marketed on each week. Traditionally, they would have an employee go through each platform and copy the relevant data. The employee would then draft a metric report with charts and tables. I automated this with Python. The fpdf and requests library were most notable in the project; requests were made to the different platform API’s, data was saved to a file, and a pdf was created with plots, figures, and images that were created from the data. This project added significant value to the company and reduced their expenses.

Anything that measures will grow. Having a system in place that regularly collects quantitative data is indispensible to a company. I was employed to create a system that would collect data from website and save the data to a file daily. I built web drivers that required extensive logic to navigate a website through many obstacles and collect data in very untraditional methods. Web scraping was done with selenium in Python and data was appended to an ongoing csv file.

It takes years for a brand to build trust and only seconds to diminish it. Having systems in place to identify catalysts that could hinder or benefit a brand’s reputation is crucial. I developed systems that would automatically collect comments on various social platforms, and using the natural language tool kit library in Python, was able to identify if a catalyst existed that could dramatically affect a company’s perception based on the polarity and magnitude of the comment.

Automation is predicted to replace over 1.5 million jobs over the next decade. If automation is not in a business’s toolkit, their competitors who employ it will soon overtake them. Automation can be done on Linux, Windows, and Macintosh systems all with Python, and can be done with ease based on the vast number of Python libraries that create solutions to many problems.
